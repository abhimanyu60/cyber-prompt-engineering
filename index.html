Segment One
Prompt Foundations
Effective AI use in security begins with a single discipline — writing instructions with the same rigour applied to a threat brief.
Foundations Why Instruction Quality Defines AI Intelligence Quality 02 / 16
The Core Principle
A prompt is the complete instruction set provided to the AI. The model has no implicit understanding of your environment, risk appetite, or priorities — it works entirely from what is written. Output quality is a direct function of instruction quality.
The Security Context
M365 Copilot has access to email, Teams, SharePoint, and Defender alerts. The same underlying capability can produce a generic response or a precisely scoped security analysis — the difference lies entirely in how the instruction is constructed.
Principle: The AI defaults to the average interpretation of any ambiguous instruction. Precision — not volume — is what drives superior security intelligence from AI.
Contrast: Unstructured vs Structured
UNSTRUCTURED — BROAD OUTPUT
Review emails for phishing.
STRUCTURED — PRECISE INTELLIGENCE
You are a SOC analyst. Review emails in the shared Security mailbox over the last 7 days. Identify messages with urgency language, external senders spoofing internal domains, or suspicious attachments. Return a table: Sender | Subject | Risk Indicator | Priority.
What Changed
The structured version defines a role, a data scope, specific detection signals, and the output format. This is the RCTF framework — covered on the next slide.
Foundations The RCTF Framework — Anatomy of a Precision Prompt 03 / 16
R · Role
Assign the AI a specific expert identity. This shapes vocabulary, confidence thresholds, and analytical lens. "You are a senior threat intelligence analyst…"
C · Context
Provide the operational situation. Which system? What timeframe? What environment? Specificity separates generic advice from actionable intelligence.
T · Task
State the precise action required. "Summarise" invites ambiguity. "Classify each alert as Critical / High / Medium / Low with one-line reasoning" leaves no room for drift.
F · Format
Specify the output structure. Table, executive paragraph, JSON — this determines whether output is immediately operationally usable or requires rework.
RCTF = Role · Context · Task · Format. Applying all four consistently is what separates reactive AI use from a structured intelligence capability.
Applied — Defender Alert Triage
RCTF IN ACTION · M365 COPILOT
[R] You are a Tier-1 SOC analyst. [C] 12 Microsoft Defender alerts fired in the last 4 hours across our Azure tenant — involving unusual sign-in geographies and token reuse patterns. [T] For each alert: determine whether this represents a likely account takeover or a probable false positive. Provide one-sentence reasoning. [F] Return a markdown table: Alert ID | Verdict | Confidence | Reasoning
Outcome
The AI knows its authority level, the data scope, the exact decision required, and the output format. The result is directly usable — no interpretation required from the analyst.
Foundations Reference Use Cases — Foundational Security Workflows 04 / 16
① Phishing Email Triage
Structured analysis of a suspected phishing email forwarded to the Security mailbox.
REFERENCE PROMPT
You are a phishing analyst. Assess this email: (1) Is the sender domain legitimate or spoofed? (2) Does the body use urgency, fear, or credential-harvesting language? (3) Are there suspicious links or attachments? Return risk rating — HIGH / MEDIUM / LOW — with a 2-sentence justification.
② Executive Security Update
Auto-draft a structured security update for CISO or committee review from weekly alerts and tickets.
REFERENCE PROMPT
You are a cybersecurity communications lead. From this week's security alerts and incident tickets, produce a 5-point executive summary covering: top threat categories observed, any active incidents, and recommended leadership actions. Language should be precise, non-technical, and action-oriented.
③ Policy Gap Assessment
Identify control gaps in a security policy document against a compliance framework.
REFERENCE PROMPT
You are an ISO 27001 compliance analyst. Review the attached security policy. Identify gaps against NIST CSF 2.0 controls. Return a table: Control ID | Gap Description | Risk if Unaddressed | Priority (High / Med / Low).

Segment Two
Intermediate Prompting
Layer operational constraints, environmental anchoring, and sequential reasoning — the techniques that establish AI as a structured analytical capability.
Intermediate Constraint Layering — Governing AI Behaviour in Security Contexts 05 / 16
Why Constraints Are Essential
AI fills ambiguous gaps with the most statistically probable interpretation — which is rarely aligned to a specific operational requirement. Constraints define the boundaries of the AI's authority, scope, and outputs, making results defensible and auditable.
C1Scope: Only use data from [system / timeframe]
C2Audience: Write for a non-technical executive audience
C3Accuracy: If data is insufficient, state that — do not infer
C4Authority: Recommend only within Tier-1 analyst authority
C5Length: Maximum 200 words / 5 bullet points
Applied — Insider Threat Behavioural Review
CONSTRAINED PROMPT · DLP ANALYSIS
You are a DLP analyst. Context: 30-day file-access log for a user who submitted resignation 2 weeks ago. Identify anomalous patterns: bulk downloads, out-of-scope data access, after-hours activity, or access to Finance / HR systems. Constraints: - Flag patterns only — do not draw conclusions on intent - Do not reference data outside this log - Note any plausible benign explanation - Output: Risk summary paragraph + evidence table
Note: The constraint "flag patterns only — do not conclude on intent" is deliberate governance — keeping AI within the boundaries of what is legally and procedurally appropriate for an initial review.
Intermediate Prompt Chaining — Sequencing Complex Security Analysis 06 / 16
The Approach
A single prompt attempting to cover extraction, enrichment, impact analysis, and communication simultaneously produces diluted results. Chaining breaks the task into discrete, verifiable steps — each prompt takes the previous output as its input. Every step can be reviewed before proceeding.
Applied Scenario — Post-Ransomware Incident Workflow
Step 1 · Extract
"Extract all IOCs (IPs, file hashes, domains) from this incident report. Output as a structured JSON list."
Step 2 · Enrich
"Map each IOC to known threat actor groups or campaigns. Assign a confidence score to each attribution."
Step 3 · Impact
"For each confirmed IOC, describe the likely business impact if the threat actor achieves its objective in a regulated financial environment."
Step 4 · Brief
"Convert this enriched intelligence into a 1-page executive brief. Tone: calm, factual, action-first. No technical jargon."
✓
Each step independently verifiable before proceeding
~2hr
Analyst time recovered per post-incident report

Full prompt chain is loggable — defensible for regulators
Intermediate Environment-Anchored Threat Intelligence 07 / 16
The Challenge
Security teams consume 15–20 threat intel feeds daily. The volume is unmanageable at analyst pace. Anchoring the AI to a specific technology stack filters out irrelevant threats and surfaces only what applies to your environment.
Environment-Anchored Prompting
Prepending your technology stack and regulatory context to every threat intel prompt causes the AI to automatically exclude threats targeting systems you don't operate — and weight its analysis against your actual exposure surface.
Recommendation: Maintain a standard Environment Profile snippet — stack, industry, crown jewels — and prepend it to every threat analysis prompt. This creates consistent, targeted intelligence output across the team.
Threat Intel Digest Prompt
ENVIRONMENT-ANCHORED · THREAT INTEL
Role: Senior Threat Intelligence Analyst Environment: - Industry: Financial Services - Stack: Azure AD, M365, SAP ERP, Linux (RHEL 8), Windows Server 2019 - Priority assets: Trading platform, Core Banking, Customer PII Task: Review the 3 attached threat bulletins. For each: 1. Does this threat target our stack? 2. What is the likely attack vector? 3. What is the potential blast radius? 4. Which single control reduces risk most? Format: One card per bulletin, max 5 lines. Urgency flag: 🔴 🟡 🟢

Advanced Chain-of-Thought — Explicit Reasoning in Security Analysis 08 / 16
The Technique
Instructing the AI to reason step-by-step before concluding produces materially different results from direct queries. The model constructs a logical chain rather than pattern-matching to a surface-level answer — critical when the reasoning itself is part of the deliverable.
Security Applications
Attack path reconstruction · Root cause investigation · Threat actor attribution · Risk quantification · Response prioritisation. Wherever the chain of logic must be auditable, chain-of-thought prompting is the standard approach.
Activation: Adding "Walk me through your reasoning step by step before concluding" consistently improves analytical depth — and produces reasoning that can be reviewed, challenged, and documented.
Applied — Attack Path Reconstruction
CHAIN-OF-THOUGHT · SIEM ANALYSIS
You are an incident response specialist. SIEM events — 6-hour window: 03:14 | VPN brute force (200 attempts) 03:47 | Successful login from TOR exit node 04:02 | Lateral movement to File Server 04:18 | Large ZIP created in Temp directory 04:31 | DNS query to unknown external domain 04:44 | SIEM alert suppressed by admin account Reason through each event in sequence: 1. Which attack stage does each event represent? 2. What is the attacker's most probable objective? 3. Which event was the critical detection failure point? 4. What control would have disrupted the chain earliest? Show your reasoning per step before your final assessment.
Advanced Few-Shot Prompting — Encoding Organisational Risk Standards into AI 09 / 16
The Concept
By providing 2–3 labelled examples before the task, the AI calibrates to your classification standards — your severity language, your risk thresholds, your organisational context — rather than its default training assumptions.
Strategic Value
Every organisation defines risk differently. A "Critical" in retail carries different weight than in a regulated financial institution. Few-shot prompting encodes your organisation's risk language into every AI output — creating consistency across teams and shifts.
Asset value: A library of 5–10 calibration examples from past incidents becomes the organisation's AI classification standard — a reusable intellectual asset across the entire security function.
Applied — Phishing Classification at Scale
FEW-SHOT · EMAIL CLASSIFICATION
Classify emails as PHISHING / SUSPICIOUS / CLEAN. Calibration examples: "Your account will be locked. Click here immediately." → PHISHING (urgency language + credential threat + generic link) "Attached Q3 invoice for review." from unknown external domain → SUSPICIOUS (unknown sender + financial document) "Team lunch at 12pm, usual spot." → CLEAN (internal, no links, social context) Classify the following emails using the same format. For each: Verdict | Primary signal driving the verdict. [PASTE EMAIL BATCH HERE]

Advanced Prompting vs Agentic AI — Choosing the Right Operating Model 12 / 16
The Strategic Distinction
Prompt-based AI is a human-in-the-loop capability — you direct, AI responds, you act on the output. Agentic AI operates autonomously across multiple steps, calling tools, processing results, and routing decisions with minimal human intervention per cycle. This choice is a governance and risk decision, not a technical one.
✦ PROMPT-BASED — Human in the Loop
→Task is discrete — one analysis, one decision, one document
→Human sign-off required before any operational action
→Input data is already available in the session context
→Analyst augmentation is the primary objective
Use cases: Email triage · Alert classification · Threat intel digest · Exec brief drafting · Policy review
⚡ AGENTIC AI — Autonomous Orchestration
→Task is high-volume and repetitive — 500 mailboxes, 200 IOCs, 30 tickets
→AI must query external systems — SIEM, threat intel APIs, ticketing platforms
→Volume makes per-task human review impractical — continuous compliance, daily log analysis
→AI needs to route and escalate — triage queue management, automated response triggers
Use cases: Phishing queue automation · IOC enrichment pipelines · Compliance monitoring · SOAR integration

Governance AI Governance in Security Operations — Policy Considerations 14 / 16
⚠ Data Classification
Copilot processes only data within its approved classification tier. The following require explicit data handling policy before use:

· Customer PII above anonymised level
· Active incident data during live response
· Vulnerability scan outputs pre-remediation
· Regulatory submission drafts

Copilot is a licensed processor — subject to the same data classification controls as any third-party service.
⚡ Human Accountability
AI output is a first-draft intelligence product — not a final decision. The following require human verification before action:

· CVE identifiers and CVSS scores
· Threat actor attribution claims
· Regulatory or legal references
· Any recommendation triggering a system change

Accountability remains with the analyst who acts on the output — not with the model that produced it.
✓ Prompt Governance
Structured prompt usage creates an auditable capability:

· Log high-value prompts for audit and reproducibility
· Version-control prompt libraries in SharePoint
· Review AI outputs before external distribution
· Refresh environment profiles quarterly

A well-maintained prompt library encodes analytical standards — reproducible and auditable under regulatory scrutiny.
Governance principle: Prompt engineering is augmented analysis — not automation. AI amplifies analytical capability. Accountability for every security decision remains with the individual who acts on the intelligence.
CLOSE Strategic Takeaways & Recommended Roadmap 15 / 16
Three Strategic Principles
01Precision over volume. RCTF — Role, Context, Task, Format — applied consistently is the foundation of a structured AI intelligence capability across security operations.
02Govern the output. Constraints and human review gates are not limitations on AI capability — they are what make AI output defensible, auditable, and operationally trustworthy.
03Build the asset. A prompt library encodes analytical standards. It is version-controlled, auditable, and compounds in value. Treat it as security infrastructure.
Recommended Roadmap

Month 1: Establish RCTF as the team standard. Identify 3 recurring workflows for structured prompt deployment — triage, weekly brief, threat intel digest.

Month 2: Stand up a shared prompt library in SharePoint. Begin version-controlling high-value prompts. Measure analyst time recovered.

Month 3: Deploy chained prompting for post-incident reporting. Feed MITRE ATT&CK output from adversarial simulation into detection engineering backlog.

Quarter 2: Evaluate agentic workflow candidates. Define governance framework and pilot with phishing triage queue automation.
The objective is not AI adoption for its own sake — it is ensuring the security function derives consistent, auditable intelligence from its AI investment, and that every analyst operates at peak analytical capability.

Advanced Prompting vs Agentic AI — Choosing the Right Operating Model 12 / 16
The Strategic Distinction
Prompt-based AI is a human-in-the-loop capability — you direct, AI responds, you act on the output. Agentic AI operates autonomously across multiple steps, calling tools, processing results, and routing decisions with minimal human intervention per cycle. This choice is a governance and risk decision, not a technical one.
✦ PROMPT-BASED — Human in the Loop
→Task is discrete — one analysis, one decision, one document
→Human sign-off required before any operational action
→Input data is already available in the session context
→Analyst augmentation is the primary objective
Use cases: Email triage · Alert classification · Threat intel digest · Exec brief drafting · Policy review
⚡ AGENTIC AI — Autonomous Orchestration
→Task is high-volume and repetitive — 500 mailboxes, 200 IOCs, 30 tickets
→AI must query external systems — SIEM, threat intel APIs, ticketing platforms
→Volume makes per-task human review impractical — continuous compliance, daily log analysis
→AI needs to route and escalate — triage queue management, automated response triggers
Use cases: Phishing queue automation · IOC enrichment pipelines · Compliance monitoring · SOAR integration

Governance AI Governance in Security Operations — Policy Considerations 14 / 16
⚠ Data Classification
Copilot processes only data within its approved classification tier. The following require explicit data handling policy before use:

· Customer PII above anonymised level
· Active incident data during live response
· Vulnerability scan outputs pre-remediation
· Regulatory submission drafts

Copilot is a licensed processor — subject to the same data classification controls as any third-party service.
⚡ Human Accountability
AI output is a first-draft intelligence product — not a final decision. The following require human verification before action:

· CVE identifiers and CVSS scores
· Threat actor attribution claims
· Regulatory or legal references
· Any recommendation triggering a system change

Accountability remains with the analyst who acts on the output — not with the model that produced it.
✓ Prompt Governance
Structured prompt usage creates an auditable capability:

· Log high-value prompts for audit and reproducibility
· Version-control prompt libraries in SharePoint
· Review AI outputs before external distribution
· Refresh environment profiles quarterly

A well-maintained prompt library encodes analytical standards — reproducible and auditable under regulatory scrutiny.
Governance principle: Prompt engineering is augmented analysis — not automation. AI amplifies analytical capability. Accountability for every security decision remains with the individual who acts on the intelligence.
CLOSE Strategic Takeaways & Recommended Roadmap 15 / 16
Three Strategic Principles
01Precision over volume. RCTF — Role, Context, Task, Format — applied consistently is the foundation of a structured AI intelligence capability across security operations.
02Govern the output. Constraints and human review gates are not limitations on AI capability — they are what make AI output defensible, auditable, and operationally trustworthy.
03Build the asset. A prompt library encodes analytical standards. It is version-controlled, auditable, and compounds in value. Treat it as security infrastructure.
Recommended Roadmap

Month 1: Establish RCTF as the team standard. Identify 3 recurring workflows for structured prompt deployment — triage, weekly brief, threat intel digest.

Month 2: Stand up a shared prompt library in SharePoint. Begin version-controlling high-value prompts. Measure analyst time recovered.

Month 3: Deploy chained prompting for post-incident reporting. Feed MITRE ATT&CK output from adversarial simulation into detection engineering backlog.

Quarter 2: Evaluate agentic workflow candidates. Define governance framework and pilot with phishing triage queue automation.
The objective is not AI adoption for its own sake — it is ensuring the security function derives consistent, auditable intelligence from its AI investment, and that every analyst operates at peak analytical capability.
